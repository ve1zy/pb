# Отчет: Лабораторная 32 - Автоматизация ревью кода

## Цель
Создать AI-ревьюера для PR, который:
- Получает diff и измененные файлы
- Использует RAG (документация + код)
- Выдает текст ревью с багами, архитектурой, рекомендациями

## Реализация

### Архитектура

```
PR opened → GitHub Action → pr_reviewer.py → RAG + анализ → review_output.md → комментарий в PR
```

### Компоненты

**1. GitHub Action** (`.github/workflows/pr-review.yml`):
- Триггер: `pull_request` (opened, synchronize, reopened)
- Получает diff
- Запускает Python скрипт
- Постит ревью как комментарий

**2. Ревьюер** (`pr_reviewer.py`):
- Парсинг diff
- Статический анализ (баги, стиль)
- Архитектурные проверки
- RAG по документации
- Генерация отчета

## Проверки

### Потенциальные баги

| Паттерн | Что ищет |
|---------|----------|
| `except:` | Без типа — скрывает ошибки |
| `print()` | Для production нужен logging |
| `password=`, `secret=`, `token=` | Hardcoded секреты |
| `TODO`, `FIXME`, `XXX` | Незавершенные задачи |
| `execute("...%s...")` | SQL injection |
| `eval()`, `exec()` | Опасные вызовы |
| Функция > 50 строк | Слишком длинная |

### Стиль кода

| Паттерн | Рекомендация |
|---------|--------------|
| `def ` без `->` | Type hints |
| Без `"""` | Docstrings |

### Архитектура

| Паттерн | Проблема |
|---------|----------|
| Нет test_* файлов | Нет тестов |
| Изменения > 300 строк | Большой файл |

### RAG

- Загружает `docs_lab21/*.md`
- Keyword-based поиск
- Показывает релевантную документацию

## Использование

### Локальный запуск

```bash
# Создать тестовый diff
git diff HEAD~1 HEAD > pr_diff.txt
git diff --name-only HEAD~1 HEAD > changed_files.txt

# Запустить ревью
python pr_reviewer.py
```

### GitHub Action

Автоматически запускается на каждый PR. Результат постится как комментарий.

## Файлы

- `.github/workflows/pr-review.yml` — GitHub Action
- `pr_reviewer.py` — скрипт ревью
- `pr_diff.txt` — тестовый diff
- `changed_files.txt` — список файлов
- `review_output.md` — результат ревью
