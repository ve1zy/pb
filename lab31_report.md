# Отчет: Лабораторная 31 - Ассистент разработчика

## Цель
Создать ассистента разработчика, который:
- Использует RAG для документации (README + docs/)
- Подключен к проекту через MCP
- Отвечает на вопросы через /help

## Реализация

### Архитектура

```
┌──────────────────┐
│   Пользователь   │
│      /help       │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│    assistant.py  │
│                  │
│  ┌────────────┐  │
│  │ RAG (TF-   │  │     ┌──────────────┐
│  │    IDF)    │──┼────▶│  mcp_git_    │
│  │            │  │     │  server.py   │
│  └────────────┘  │     │  (git_branch)│
│                  │     └──────────────┘
└──────────────────┘
         │
         ▼
┌──────────────────┐
│  README + docs/  │
│   (9 документов) │
└──────────────────┘
```

### Компоненты

**1. RAG-система:**
- Документы: README.md + docs_lab21/*.md (9 файлов)
- Метод: TF-IDF (656 слов в словаре)
- Чанки: 9 (по 500 слов)
- Retrieval: косинусное сходство

**2. MCP-сервер (`mcp_git_server.py`):**
- Инструменты:
  - `git_branch` — текущая ветка + список всех
  - `list_files` — файлы по расширению
  - `git_diff` — diff последних N коммитов
- Протокол: MCP (Model Context Protocol)

**3. Ассистент (`assistant.py`):**
- Команды: `/help`, `/git`, `/files`, `/exit`
- RAG для ответов на вопросы
- MCP для git-операций
- Fallback на описание структуры проекта
