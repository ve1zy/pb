## Компоненты

### 1. FAQ (`docs_lab33/faq.md`)
- 14 вопросов-ответов
- Категории: авторизация, биллинг, API, аккаунт, известные проблемы

### 2. Тикеты (`tickets.json`)
- 4 тикета от 3 разных пользователей
- Категории: auth, api, performance
- История сообщений в каждом тикете

### 3. MCP-сервер (`mcp_support_server.py`)
- `get_ticket(ticket_id)` — получить тикет
- `list_tickets(user_email, status)` — список тикетов
- `search_tickets(query)` — поиск по тикетам
- `add_ticket_message(ticket_id, text, from_user)` — добавить сообщение

### 4. Ассистент (`support_assistant.py`)
- RAG по FAQ (TF-IDF)
- MCP-клиент для тикетов
- Учёт контекста пользователя
- Команды: /help, /faq, /tickets, /ticket T-XXX

## Результаты тестирования

### Сценарий 1: "Почему не работает авторизация?"

```
💼 Ваши открытые тикеты:
📋 T-001 - Не работает авторизация (high)

📖 Ответ из базы знаний:
**В: Почему не работает авторизация?** (score: 0.29)
О: Проверьте:
1. Правильность email и пароля
2. Не заблокирован ли аккаунт
3. Подтверждён ли email
4. VPN из запрещённой страны
```

✅ RAG нашёл правильный ответ + показал тикет пользователя

### Сценарий 2: "Ошибка 429"

