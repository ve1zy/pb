## Компоненты

### 1. FAQ (`docs_lab33/faq.md`)
- 14 вопросов-ответов
- 5 категорий: авторизация, биллинг, API, аккаунт, баги

### 2. Тикеты (`tickets.json`)
- 4 тикета от 3 пользователей
- Категории: auth, api, performance
- История сообщений

### 3. MCP-сервер (`mcp_support_server.py`)
- `get_ticket(ticket_id)`
- `list_tickets(user_email, status)`
- `search_tickets(query)`
- `add_ticket_message(ticket_id, text, from_user)`

### 4. Ассистент (`support_assistant.py`)
- **RAG** по FAQ (TF-IDF)
- **MCP-клиент** для тикетов
- **LLM** (TinyLlama 1.1B) для:
  - Генерация естественных ответов (`llm_make_natural`)
  - Резюме тикета (`llm_summarize_ticket`)
  - Рекомендации оператору (`llm_suggest_actions`)

## Команды ассистента

| Команда | Описание |
|---------|----------|
| `/help` | Справка |
| `/faq` | Показать FAQ |
| `/tickets` | Открытые тикеты пользователя |
| `/ticket T-001` | Тикет + LLM-резюме |
| `/analyze T-001` | Анализ + LLM-рекомендации |
| `<вопрос>` | RAG + LLM + контекст |

## Результаты тестирования

### Сценарий 1: "Почему не работает авторизация?"

```
💼 Ваши открытые тикеты:

🤖 Ответ ассистента (LLM):

```

## Технические детали

**LLM:** TinyLlama-1.1B-Chat (GGUF Q4_K_M)
- Lazy loading — загружается при первом использовании
- Генерирует естественные ответы, резюме, рекомендации
- Graceful fallback — если LLM недоступна, работает без LLM-блоков

**RAG:** TF-IDF по 14 Q&A блокам
- Находит правильные ответы в FAQ
- Возвращает score для оценки релевантности

**MCP:** JSON-RPC через stdio_client
- Читает/пишет tickets.json
- 4 инструмента для работы с тикетами
