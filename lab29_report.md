# Отчет: Лабораторная 29 - Оптимизация локальной LLM

## Цель
Оптимизировать локальную модель TinyLlama 1.1B под задачу RAG:
- Настроить параметры (temperature, max_tokens)
- Протестировать разные квантования

## Тестовая конфигурация

**Модель:** TinyLlama-1.1B-Chat (GGUF)
**Задача:** RAG-система (ответы на вопросы по документации)
**Тестовые запросы:**
1. Что такое Docker и как его использовать?
2. Как работает asyncio в Python?
3. Что такое MCP протокол?

## Рекомендации по оптимизации

### 1. Temperature
- **0.1-0.3** — фактологические вопросы, RAG
- **0.5-0.7** — общий диалог
- **0.8-1.0** — креативные задачи

### 2. Max tokens
- **50-100** — краткие ответы (определения)
- **100-200** — средние ответы (объяснения)
- **200-500** — длинные ответы (эссе, код)

### 3. Квантование
- **Q2_K** — максимум скорости (прототипы)
- **Q4_K_M** — баланс (рекомендуется)
- **Q5_K_M/Q8_0** — максимум качества

### 4. Prompt-шаблон
- Четкие инструкции (что делать, чего не делать)
- Ограничение контекстом (используй ТОЛЬКО...)
- Формат ответа (кратко, по пунктам, и т.д.)
- Системная роль (ты - эксперт по...)

### 5. Контекст
- Ограничить до 200-300 слов
- Удалить нерелевантную информацию
- Структурировать (заголовки, списки)

## Практические советы

### Для RAG-систем:
```python
# Оптимальные параметры

# Prompt-шаблон
```

```python

prompt = """Ты - дружелюбный ассистент.

ИСТОРИЯ:
{history}

ОТВЕТ:"""
```

## Ограничения

1. **Маленькая модель (1.1B)** — не подходит для сложных задач
2. **Контекст 512 токенов** — ограничивает RAG
3. **Русский язык** — качество ниже, чем у английских моделей

**Решение:** Использовать более мощные модели:
- Llama 2 7B/13B
- Mistral 7B
- Qwen 7B/14B
