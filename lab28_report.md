# Отчет: Лабораторная 28 - Локальная LLM + RAG

## Цель
Создать полностью локальную RAG-систему, используя:
- Индекс из lab21 (TF-IDF векторизация)
- Локальную LLM (TinyLlama 1.1B через ctransformers)
- Сравнение с облачной моделью (если доступно)

## Реализация

### Архитектура RAG-пайплайна

```
Query → Retrieval (TF-IDF) → Top-K чанков → Context → LLM → Answer
```

### Компоненты

1. **Retrieval** (локальный):
   - Индекс: `index_lab21.json` (TF-IDF матрица 8×626)
   - Словарь: 626 слов
   - Чанков: 8 (из 8 документов)
   - Метрика: косинусное сходство

2. **Генерация** (локальная):
   - Модель: TinyLlama-1.1B-Chat (GGUF Q4_K_M)
   - Библиотека: ctransformers
   - Контекст: до 300 слов (~512 токенов)
   - Параметры: temperature=0.7, max_tokens=150

3. **Генерация** (облачная):
   - Модель: GPT-3.5-turbo (OpenAI)
   - Требует: OPENAI_API_KEY
   - Статус: не тестировалась (ключ не установлен)

## Проблемы и ограничения

### 1. Маленькая модель (1.1B параметров)
- Недостаточно для качественного понимания контекста
- Генерирует бессвязный текст
- Плохо работает с русским языком

**Решение:** Использовать более мощные модели:
- Llama 2 7B/13B
- Mistral 7B
- Qwen 7B/14B

### 2. Ограниченный контекст (512 токенов)
- Контекст + вопрос превышают лимит
- Предупреждения о превышении

**Решение:**
- Сокращать контекст до 200-300 слов
- Использовать модели с большим контекстом (2048-4096 токенов)

### 3. Качество retrieval
- TF-IDF не всегда находит релевантные чанки
- Низкие scores для нерелевантных документов

**Решение:**
- Использовать эмбеддинги (sentence-transformers)
- Hybrid search (TF-IDF + embeddings)
