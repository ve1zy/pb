# Lab 35: AI Release Assistant

## Задача

Автоматизировать процесс создания релизов:
- Анализировать коммиты с момента последнего тега
- Генерировать changelog с категоризацией (Features / Bug Fixes / Changes)
- Предлагать следующую версию по semver
- Создавать GitHub Release через `gh` CLI

## Проблема

Ручное создание релиза занимает время:
1. Найти последний тег
2. Просмотреть все коммиты
3. Написать changelog
4. Определить версию (major/minor/patch)
5. Создать релиз на GitHub

AI автоматизирует шаги 2-4, оставляя финальное подтверждение за человеком.

## Как AI участвует

### 1. Генерация changelog
LLM получает список коммитов и категоризирует их:
- **Features** — коммиты с `feat`, `add`, `new`
- **Bug Fixes** — коммиты с `fix`, `bug`, `resolve`
- **Changes** — коммиты с `refactor`, `update`, `improve`
- **Other** — остальное

### 2. Предложение версии
LLM анализирует changelog и предлагает semver:
- MAJOR — breaking changes
- MINOR — новые фичи
- PATCH — баг-фиксы

### 3. Форматирование release notes
LLM генерирует Markdown для GitHub Release.

## Использование

### Dry run (preview)
```bash
python demo35.py
```

### Создать реальный релиз
```bash
python -c "import asyncio; from release_assistant import run_release_pipeline; asyncio.run(run_release_pipeline(dry_run=False))"
```

## Требования

- Python 3.10+
- `pip install mcp ctransformers`
- `gh` CLI (GitHub CLI) с авторизацией
- Git репозиторий с тегами

## Преимущества

1. **Экономия времени** — автоматическая генерация changelog
2. **Консистентность** — единый формат release notes
3. **Умное версионирование** — AI предлагает версию на основе изменений
4. **Dry run** — можно preview перед созданием
5. **Локальная LLM** — работает без интернета, приватно

## Ограничения

- TinyLlama 1.1B — маленькая модель, может делать ошибки в категоризации
- Требует `gh` CLI для создания релизов
- Работает только с Git репозиториями
- Нужны теги для определения "что изменилось"

## Возможные улучшения

1. Интеграция с Conventional Commits для более точной категоризации
2. Автоматическое определение breaking changes
3. Генерация migration guides для MAJOR версий
4. Отправка уведомлений в Slack/Telegram после релиза
5. Использование большей модели (Llama 3, Mistral) для качества
